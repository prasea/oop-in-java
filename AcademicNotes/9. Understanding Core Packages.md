
# Unit 9 Understanding Core Packages 

## `java.lang` and `java.util`

A **package** in Java is a **namespace** that groups related classes and interfaces together.

### Advantages of Packages

* It provides logical grouping of classes
* Avoids naming conflicts
* Improves reusability and maintainability

# `java.lang` PACKAGE


The **`java.lang` package** contains classes that are **fundamental to the Java programming language**.

* They get automatically imported in every Java program
* Contains core classes such as `Object`, `Math`, wrapper classes, and `String`

## Wrapper Classes

* Java has **primitive data types**: `byte, short, int, long, float, double, char, boolean`.
* These **primitive types are not objects**.
* Java is an **object-oriented language**, so to treat primitives as objects, Java provides **wrapper classes**.
* Wrapper classes **wrap (box)** the primitive value inside an object.


### Need for Wrapper Classes

* Primitives Data Type:
  * Have no methods
  * Are not objects
  * Cannot be used in collections like `ArrayList<Integer>` (collections store *objects*)

Wrapper classes solve this by:

* Providing an **object representation** of each primitive.
* Allowing method access on values (e.g., `Integer.parseInt()`).
* Allowing use in APIs or collections that require objects.

### Primitive vs Wrapper Mapping

Each primitive data type has a corresponding wrapper class (in **java.lang** package):

| Primitive | Wrapper Class |
| --------- | ------------- |
| byte      | Byte          |
| short     | Short         |
| int       | Integer       |
| long      | Long          |
| float     | Float         |
| double    | Double        |
| char      | Character     |
| boolean   | Boolean       |

All wrapper classes are in **java.lang**, so no import is needed.

## `Number` Class

### Definition

**`Number`** is an **abstract superclass** for numeric wrapper classes such as `Integer`, `Float`, `Double`, etc.
Common methods defined in `Number` and inherited by numeric wrappers:

* `byteValue()`
* `shortValue()`
* `intValue()`
* `longValue()`
* `floatValue()`
* `doubleValue()`


### Example


```java
public class WrapperDemo {
  public static void main(Striong[] args) {
    Number number = 100.0;
    int x = number.intValue();
  }
}
```

## Vector Class

The Vector class in Java is a **dynamic array** of objects that is part of the java.util package and implements the List interface. A key characteristic is that it is thread-safe due to its synchronized methods, unlike its more common counterpart, the ArrayList. 

### Characteristics

* Synchronized (thread-safe)
* Stores objects
* Legacy class

### Common Methods

* `addElement()`
* `removeElement()`
* `size()`

### Example

```java
Vector vector = new Vector();
  vector.add("Apple");
  vector.add("Banana");
  vector.add(0, "Grape");

  // Print and access elements
  System.out.println("Vector: " + vector);
  System.out.println("Element at index 1: " + vector.get(1));

  // Remove an element
  vector.remove("Banana");
  System.out.println("Vector after removing Banana: " + vector);

  // Iterate through elements
  System.out.println("Elements:");
  for (Object item : vector) {
    System.out.println(item);
}
```

## Stack Class

The Stack class in Java, found in the java.util package, implements a **last-in, first-out (LIFO)** data structure.  It extends the Vector class and provides methods for common stack operations

### Stack Operations

* `push()` → Insert
* `pop()` → Remove
* `peek()` → View top
* `empty()` → Check empty


```java
Stack stack = new Stack();
stack.push("A");
stack.push("B");
System.out.println("Top: " + stack.peek()); // Output: B
System.out.println("Popped: " + stack.pop()); // Output: B
System.out.println("Empty: " + stack.empty()); // Output: false
```

## Dictionary Class

The Dictionary class in Java is an **abstract class** from the java.util package, introduced in Java 1.0 to represent a collection of **key-value pairs** where each key is unique.  It serves as the parent class for Hashtable, which is its only direct known subclass

* It is Legacy class
* Replaced by `Map` interface. Recommended to use `HashMap`

### Important Methods

* `put()`
* `get()`
* `keys()`
* `elements()`

```java 
Dictionary dict = new Hashtable();
dict.put(101, "Sydney");
dict.put(102, "Perth");
System.out.println(dict.get(101)); 
```
---

## Hashtable Class

**`Hashtable`** is a **synchronized collection** that stores data in **key-value pairs**.

### Characteristics

* Does not allow `null` key or value
* Thread-safe
* Legacy class

### Example

```java
// Hashtable numbers = new Hashtable();
Hashtable<String, Integer> numbers = new Hashtable<>();
numbers.put("one", 1);
numbers.put("two", 2);
System.out.println(numbers.get("two"));
```

## Enumeration Interface

### Definition

**`Enumeration`** is used to **traverse elements** of legacy collections.

### Methods

* `hasMoreElements()`
* `nextElement()`



```java
Vector v = new Vector(List.of(1, 2, 3, 4, 5));
Enumeration e = v.elements();
while (e.hasMoreElements()) {
  System.out.println(e.nextElement());
}
```

## 15. Random Number Generation

The `java.util.Random` class in Java is a standard utility for generating pseudorandom numbers.

### Common Methods

* `nextInt()`
* `nextInt(n)`
* `nextDouble()`


```java
Random rand = new Random();
int diceRoll = rand.nextInt(6) + 1;
double randomDouble = rand.nextDouble();
System.out.println("Dice roll: " + diceRoll);
System.out.println("Random double: " + randomDouble);
```
